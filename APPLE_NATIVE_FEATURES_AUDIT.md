# Apple-Native Feature Audit

Last updated: 2026-06-03

## Product Rule

Use Apple-native features first when the job is Apple-to-Apple transfer, device continuity, sharing, camera input, shortcuts, or system-level automation.

Use Desk Agent custom bridge features only when the payload needs agent context: Hermes conversation state, live voice sessions, tool routing, approvals, Notch state, or remote/away behavior that Apple APIs do not provide.

## Current Decisions

| Need | Apple-first answer | Desk Agent custom answer | Current status |
| --- | --- | --- | --- |
| Send screenshots/files from Mac to Apple devices | AirDrop via `NSSharingService` | None needed | Done for shelf batches: `AirDrop latest` and `AirDrop batch` |
| Send selected iPhone text/image into agent context | Share sheet or app action should feed app bridge | AirSend queue to Hermes draft / Notch shelf | First native Share Extension done for text, links, and single images; AirSend is agent-context import, not AirDrop replacement |
| Continue the same chat/thread across Mac and iPhone | Handoff / `NSUserActivity` | shared `conversationId` bridge | Bridge exists; Mac advertises Hermes conversation activity and iPhone queues it into Chat with the handed-off conversation id; physical Handoff proof still needed |
| Invoke app actions from Siri/Shortcuts/widgets/Action Button | App Intents + App Shortcuts | helper action runner behind intents | First pass done on Mac and iPhone; deeper Talk/status intents still worth auditing |
| Glanceable phone state | WidgetKit / Live Activities | helper status endpoint | Widget exists; Live Activity is future for active Talk session |
| Quick diagnostics/share proof | Share sheet / `ShareLink` | diagnostics text generated by app | Done on iPhone diagnostics |
| Capture photo/document from iPhone to Mac note | Continuity Camera | custom camera only if agent vision needs live context | Candidate for Notes, not Chat/Home |
| Use iPhone as Mac camera/mic context | Continuity Camera / AVFoundation continuity devices | provider live audio bridge for Hermes Talk | Candidate; avoid rebuilding if Apple camera source works |
| Nearby local discovery | Bonjour / Network framework | helper HTTP contract | Done for local helper discovery |
| Nearby peer presence or small payloads | Multipeer Connectivity | helper fallback | Keep as secondary, not main transport |
| Cross-device clipboard | Universal Clipboard / pasteboard | explicit bridge only for agent routing | Use native clipboard where possible |
| iCloud note/file persistence | iCloud Drive / CloudKit / document picker | local app storage only for runtime state | Sidecar Vault and quick notes now default to the iCloud Obsidian `1note` vault; CloudKit later only if needed |
| Search local captured notes/items | Spotlight/Core Spotlight | helper search later | Candidate after note/media model stabilizes |
| Preview/open attachments | Quick Look / native document preview | custom preview only for agent-specific cards | Done for shelf batches, Sidecar Mac files, and embedded chat local image attachments; candidate for popout/note attachments |
| Browse/manage Mac files | Finder, Quick Look, Trash, pasteboard, native file dialogs | agent context attach/approved automation | Sidecar Mac tab is a guarded Finder-style layer; permanent delete/overwrite stay out until explicit confirmation model exists |
| System settings and permissions | System Settings deep links | none | Sidecar System launches core settings panes for Privacy, Accessibility, Screen Recording, Sound, Displays, and Network |
| Calendar/date commitments | Apple Calendar/EventKit | Hermes planning/routing | First read-only Home agenda wired on Mac; create/edit should wait for explicit confirmations |
| Contacts/people context | Contacts framework / native Contacts app | Hermes relationship memory only when needed | Native Contacts launcher wired in Actions/System; read access should wait for a clear use-case |
| Apple Watch access | App Intents, Shortcuts, widgets/complications | bridge only for agent conversation/action routing | Future target; do not start a separate watch product |
| Health / activity / sleep | HealthKit on iPhone/watchOS | iPhone companion summary into Hermes/Notch | Future iPhone app target; do not chase direct Mac HealthKit access |

## What Not To Duplicate

- Do not build a custom AirDrop clone for normal files.
- Do not make AirSend a generic file transfer product.
- Do not rebuild system-level device continuity like Universal Control; design around it and document assumptions.
- Do not add a second Mac companion surface just because Apple has a matching API.

## Near-Term Integration Targets

1. **Handoff for conversation continuity**
   - Done on Mac: Notch advertises a legacy MarkShot Handoff activity id with current `conversationId`, module, and surface.
   - Done on iPhone: app declares the same activity type, adopts the handed-off `conversationId` for the next phone turn, and routes continuation into the shared Hermes conversation surface.
   - Next: physical Mac-to-iPhone Handoff proof.
   - Goal: start on iPhone, continue in Notch, or vice versa, without a new visible surface.

2. **Share extension / Share sheet intake**
   - From iPhone Photos, Files, Safari, or selected text: share into Desk Agent.
   - Route options:
     - normal file transfer -> system AirDrop / Files
     - agent context -> Hermes draft, Notch shelf, or Obsidian note
   - Done: iPhone `Send to Hermes` Share Extension sends text, links, and single images into the existing agent-context queue.
   - Next: physical Share Sheet proof from Photos/Safari and decide whether Notes should receive its own share destination.

3. **Continuity Camera in Notes**
   - Notes is the right place for "scan/take photo and attach to note."
   - Keep it separate from Talk unless the camera is being used for live agent context.

4. **App Intents parity on Mac and iPhone**
   - Done on Mac: Open Desk Agent, Show Shelf, Talk to Hermes, Capture Region, Record Clip, Save Desk Agent Note, AirDrop Latest Shelf Batch, Check Desk Agent Status.
   - Done on iPhone: Talk/status/launch intents exist.
   - Next: richer status/readiness wording after the bridge capability model stabilizes.

5. **Quick Look for media attachments**
   - Done: shelf batches and embedded chat local image attachments can preview through native Quick Look.
   - Done: Sidecar Mac tab can Quick Look selected files.
   - Next: extend native previews to popout/note attachments when those attachment models stabilize.

6. **Apple Calendar and Contacts**
   - Done on Mac: Home can request Calendar access and show upcoming Apple Calendar events for the next 7 days.
   - Done on Mac: Actions/System can launch Calendar, Reminders, and Contacts as native Apple apps.
   - Calendar should remain EventKit first for date/time commitments, reminders, and schedule checks.
   - Contacts should be native Contacts first for people lookup, not a custom people database.
   - Next: add Contacts read-only lookup only if Hermes needs people context; create/edit must wait for explicit confirmations.

7. **Apple Watch**
   - Use App Intents/Shortcuts as the first path so Watch, Action Button, Siri, and widgets can trigger the same Desk Agent actions.
   - Keep Hermes/bridge routing behind those intents.
   - Next: define the smallest Watch surface: Talk to Hermes, Check Status, Save Note, maybe Capture/Share proof.

8. **HealthKit**
   - Health/activity/sleep should come through the iPhone app or Watch-linked iPhone permissions.
   - Mac Notch can display a daily summary after the phone app syncs it.
   - Do not build a fake Mac-side Health reader.
